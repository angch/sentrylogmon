## 2025-05-15 - Fast-path byte sequence rejection before JSON parsing
**Learning:** In log processing hot paths (like `JsonDetector`), parsing JSON for every single log line is a massive performance bottleneck. The `JsonDetector` extracts a `field` and uses a regex to match the value, but it was unmarshaling JSON on every single line, even if the line didn't contain the target key.
**Action:** Implemented a fast-path byte sequence rejection (e.g., `bytes.Contains` in Go, `line.windows().any()` in Rust, `std.mem.indexOf` in Zig) before heavy JSON parsing to dramatically reduce overhead for non-matching lines. Ensure this optimization is replicated across all language ports (Go, Rust, Zig) to maintain performance parity.
