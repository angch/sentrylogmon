## 2026-03-05 - Insecure Command Splitting with strings.Fields
**Vulnerability:** Secrets in command arguments were partially exposed in status output because `strings.Fields` splits quoted strings containing spaces (e.g., `--password "secret code"` -> `["--password", "\"secret", "code\""]`), causing the sanitizer to miss the second part of the secret.
**Learning:** Naive string splitting functions like `strings.Fields` or `strings.Split` are insufficient for parsing command-line arguments that may contain quotes or escapes. This leads to both functionality bugs (commands failing) and security issues (failed redaction).
**Prevention:** Always use a proper argument parser (like `google/shlex` or a custom implementation like `SplitCommand`) when handling command strings that might contain user input or configuration.
